Pipeline previsionale

Run offline BioAnalyst -> cache forecast -> dashboard

Cruscotto tecnico per ripassare il forecast BioMap: batch nativi, modello BioAnalyst, rollout, export cell_matrix, backend cache e UI in modalita previsione.

Report completo: REPORT_PIPELINE_PREVISIONALE.md. Workflow sorgente: workflow_previsionale.mmd.

Offline Il modello gira via script, non dal browser.
Cache La UI legge previsioni/YYYY-MM/cell_matrix.
Forecast UI Proiezione futura, non backtest con metriche errore.
2 Runner operativi: run.py e run_rollout.py.
9 Variabili lette dal cache forecast in dashboard.
6 Mesi target default: aprile-settembre 2026.
0 Run live dal frontend.

Workflow operativo diviso in due meta

Produzione offline

run.py / run_rollout.pyEntry point operativo per one-step e rollout.
bioanalyst_model_utils.pyCostruisce batch `.pt` da BioCube e config modello.
bioanalyst_native_utils.pyCarica runtime BioAnalyst ed esegue one-step o rollout.
native pt + manifestOutput nativo ispezionabile e validabile.
previsioni/YYYY-MM/cell_matrixWorkbook full_grid pubblicati per il backend.

Lettura online

biomap-dashboard.tsxToggle Previsione e mese target.
app/api/forecastProxy Next.js verso FastAPI.
POST /api/forecastLegge cache, ritaglia bounding box, aggrega mesi.
forecast cells ParquetCelle forecast per la mappa tematica.
Mappa, trend, tabellaRendering forecast senza riesecuzione modello.

Checklist di ripasso

La dashboard forecast non lancia BioAnalyst.
Il primo mese e one-step; i successivi sono rollout.
Il backend legge workbook, non file `.pt`.
La temperatura va normalizzata se arriva in Kelvin.
Le metriche errore restano fuori dalla UI forecast.
NDVI e rollout lungo vanno interpretati con cautela.
