Pipeline osservativa

BioCube -> Python -> FastAPI -> Next.js

Cruscotto tecnico per ripassare la catena osservativa BioMap: selezione area, lettura dati reali, calcolo cella-mese, aggregazione area-mese e rendering dashboard.

Sorgente principale: TCBiomap/tesi_bioanalyst_repo_native. Report completo: REPORT_PIPELINE_OSSERVATIVA.md.

Modalita Observed only, nessun forecast nella vista principale.
Output CSV, CSV Excel-friendly, XLSX, summary JSON, Parquet celle.
UI Mappa, trend, tabella mensile, export tabella visibile.
9 Variabili area/cella principali in dashboard.
6 Endpoint FastAPI/Next coinvolti nella vista osservativa.
1 Script osservativo orchestratore: selected_area_indicators.py.
0 Fallback demo: senza backend reale la UI mostra errore.

Workflow operativo

Frontend

biomap-dashboard.tsxArea, periodo, modalita osservazione, variabile selezionata.
app/api/indicatorsProxy Next.js verso FastAPI.
indicator-map / trend-chartRender cella-mese e area-mese.

Backend

POST /api/indicatorsValida body e sorgenti BioCube.
build_indicator_commandCostruisce CLI con area, periodo e output mode.
run_indicator_jobEsegue subprocess e rilegge output.

Script Python

selected_area_indicators.pyOrchestratore osservativo end-to-end.
build_selected_cell_month_frameTabella per cella, mese e variabili osservate.
compute_area_climate_monthlyAggregazione pesata area-mese.

Dati e output

BioCubeERA5, NDVI, soil, agriculture, forest, species.
outputs/local_previewCSV, XLSX, JSON, Parquet.
